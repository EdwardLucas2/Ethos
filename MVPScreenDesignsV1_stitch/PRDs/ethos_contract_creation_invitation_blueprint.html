# Ethos: Contract Creation & Invitation System This document defines the two-phase flow for bringing a multiplayer habit contract to life: **Phase 1: Initiation** (The Creator) and **Phase 2: Commitment** (The Invitees). --- ## 1. Phase 1: Initiation (The Creator's Flow) **Role:** The person who kicks off the challenge. **Responsibilities:** Setting the "Ground Rules" for everyone. ### User Stories * "As a creator, I want to name the contract so we have a shared identity (e.g., 'The Gym Rats')." * "As a creator, I want to set the start date and frequency period (Weekly, Bi-weekly, Monthly) so we're all on the same timeline." * "As a creator, I want to invite my friends from my contact list so I can challenge the right people." * "As a creator, I want to define the shared forfeit (e.g., 'A Pint') so everyone knows what's at stake." * "As a creator, I want to set my own personal habit and frequency immediately so my friends see my commitment when they join." ### Screen Anatomy 1. **Contract Basics:** Input fields for Name and Forfeit. 2. **Timing Controls:** Segmented controls for Period (Week/2-Week/Month) and a Date Picker for the Start Date. 3. **Friend Selector:** A search-and-add interface for finding and tagging friends. 4. **Creator's Commitment:** A mini-form within the screen to set their own goal (e.g., "Gym" @ "3x"). --- ## 2. Phase 2: Commitment (The Invitee's Flow) **Role:** The people invited to the challenge. **Responsibilities:** Defining their personal contribution to the group goal. ### User Stories * "As an invitee, I want to see a clear summary of the contract (the name, the duration, the forfeit, and who else is in) before I agree to join." * "As an invitee, I want to see what my friends have committed to so I can choose a habit of similar difficulty." * "As an invitee, I want to set my own unique habit and frequency within the creator's time period." * "As an invitee, I want to 'Lock In' my commitment to officially join the contract." ### Screen Anatomy 1. **The Invitation Header:** A high-contrast card showing "ALEX CHALLENGED YOU TO: THE GYM RATS". 2. **Group Stats Card:** A summary of the shared forfeit, start date, and cycle duration. 3. **The Commitment Form:** A prominent input area for the invitee to type their habit and select their frequency (within the constraints of the period). 4. **"Lock It In" CTA:** The primary action that transitions the user from an invitee to an active participant. --- ## 3. The "Lock In" Mechanic The contract does not move to the "Active" state until: 1. The start date is reached. 2. At least two people (Creator + 1 Invitee) have "Locked In" their commitments. 3. The Creator gives a final confirmation (if they choose to wait for more people). --- ## 4. Visual Language Requirements * **The "Draft" State:** Use dashed borders or lighter shadows for sections that haven't been finalized yet. * **Urgency:** Use the primary 'Action Blue' or 'Warning Red' for the "Lock It In" buttons. * **Clarity:** Use chunky progress-bar-style icons to represent the frequency selection (e.g., "3/7" blocks).
