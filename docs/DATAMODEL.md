# Data Model

PostgreSQL database. Migrations live in `backend/db/migrations/` ‚Äî create with `cd backend && dbmate new <description>`.

## Conventions

- All primary keys are `UUID DEFAULT gen_random_uuid()` ‚Äî never auto-increment integers.
- All timestamps are `TIMESTAMPTZ`.
- **Timezones (MVP):** All date logic runs in UTC. `DATE` columns (`cycles.start_date`, `cycles.end_date`, `contracts.start_date`) are interpreted as UTC midnight. The background scheduler uses UTC when evaluating cycle transitions. **Future:** add `timezone TEXT NOT NULL DEFAULT 'UTC'` (IANA format, e.g. `'Europe/London'`) to `contracts` and convert cycle date columns to `TIMESTAMPTZ` storing the UTC instant of local midnight.
- Booleans use `BOOLEAN`, not integers.
- Prefer `TEXT` with `CHECK` constraints over PostgreSQL enums ‚Äî easier to extend without migrations.
- Prefer explicit columns over JSON blobs.
- Foreign keys use `ON DELETE CASCADE` only where child rows have no meaning without the parent (e.g. `device_tokens`). All other FK deletions are handled at the application layer.

## Object Storage

Files are not stored in the database. A `UUID` is generated by the service, stored in the DB column, and the storage key is derived by convention:

- `users.avatar_id` ‚Üí `avatars/{avatar_id}`
- `evidence.photo_id` ‚Üí `evidence/{photo_id}`

The `FileStorageService` interface resolves keys to URLs. See `TECHSTACK.md` for backend and storage details.

## Tables

### `users`

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`supertokens_user_id`** `TEXT NOT NULL UNIQUE` ‚Äî reference to SuperTokens auth system; never exposed in API responses
- **`display_name`** `TEXT NOT NULL` ‚Äî free text, shown throughout UI
- **`tag`** `TEXT NOT NULL UNIQUE` ‚Äî short lowercase alphanumeric handle, auto-generated (e.g. `edward4f2a`); used for friend search
- **`email`** `TEXT NOT NULL UNIQUE`
- **`avatar_id`** `UUID` ‚Äî nullable; resolves to `avatars/{avatar_id}` in object storage
- **`created_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`

Tag generation (service layer): take the first word of `display_name`, lowercase, strip non-alphanumeric, truncate to 8 chars, append 4 random alphanumeric chars. Retry with a new suffix on collision.

Indexes: B-tree on `tag` ‚Äî supports `LIKE 'prefix%'` prefix search efficiently since tags are stored lowercase.

### `device_tokens`

Stores Expo push tokens for push notification delivery. One user may have multiple rows (one per device).

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`user_id`** `UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE`
- **`token`** `TEXT NOT NULL UNIQUE` ‚Äî Expo push token, format: `ExponentPushToken[xxx]`
- **`platform`** `TEXT NOT NULL CHECK (platform IN ('ios', 'android'))`
- **`created_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`
- **`last_seen_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()` ‚Äî updated on each app open; used to prune stale tokens

When Expo returns `DeviceNotRegistered` for a token, delete the row. Fan out to all tokens for a user when sending a push.

Indexes: `user_id`.

---

### `contacts`

One-way contacts list. Alice adding Bob does not imply Bob has Alice in his contacts. No approval required. Used to populate quick-add suggestions in the contract builder. Users not in your contacts are still findable by tag search and invitable to contracts.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`user_id`** `UUID NOT NULL REFERENCES users(id)` ‚Äî the user who added the contact
- **`contact_user_id`** `UUID NOT NULL REFERENCES users(id)` ‚Äî the user who was added
- **`created_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`

Constraints: `UNIQUE (user_id, contact_user_id)`, `CHECK (user_id != contact_user_id)`.

Indexes: `user_id`.

---

### `contracts`

The top-level accountability agreement. Created in `draft` status during the lobby phase before any cycles exist.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`creator_id`** `UUID NOT NULL REFERENCES users(id)` ‚Äî retained after contract starts; drives lobby powers (remove participants, start contract)
- **`name`** `TEXT NOT NULL`
- **`forfeit`** `TEXT NOT NULL` ‚Äî free text description of what losers owe, e.g. "A pint"
- **`period`** `TEXT NOT NULL CHECK (period IN ('weekly', 'biweekly', 'monthly'))`
- **`start_date`** `DATE NOT NULL` ‚Äî set in lobby; first cycle begins on this date
- **`status`** `TEXT NOT NULL DEFAULT 'draft' CHECK (status IN ('draft', 'active', 'ended', 'cancelled'))`
- **`created_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`

Status transitions:

- `draft` ‚Üí `active`: creator starts contract, ‚â•2 participants signed, start date reached
- `draft` ‚Üí `cancelled`: start date reached with <2 signed participants
- `active` ‚Üí `ended`: all remaining participants opt out after a cycle settles

---

### `participants`

A user's membership and commitment within a contract. The creator has a participant row created at contract creation with `sign_status = 'drafting'`; they sign explicitly via the lobby before starting the contract. `participants.id` is the contract-scoped identity used throughout (evidence, votes) ‚Äî prefer this over `user_id` when operating within contract context.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`contract_id`** `UUID NOT NULL REFERENCES contracts(id)`
- **`user_id`** `UUID NOT NULL REFERENCES users(id)`
- **`habit`** `TEXT` ‚Äî nullable until entered; locked when signed
- **`frequency`** `INTEGER CHECK (frequency > 0)` ‚Äî number of habit actions per cycle; nullable until set; locked when signed
- **`sign_status`** `TEXT NOT NULL DEFAULT 'waiting' CHECK (sign_status IN ('waiting', 'drafting', 'signed', 'declined', 'removed'))` ‚Äî `declined`: invitee chose not to join; `removed`: creator kicked them from the lobby. Rows are **never deleted** ‚Äî set the status instead. All queries counting active participants must filter `WHERE sign_status NOT IN ('declined', 'removed')`.
- **`opted_out_of_next_cycle`** `BOOLEAN NOT NULL DEFAULT false` ‚Äî set on last day of a cycle; service excludes this participant when creating the next cycle; never reset (opt-out is permanent removal)
- **`invited_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`
- **`signed_at`** `TIMESTAMPTZ` ‚Äî nullable until signed

Constraints: `UNIQUE (contract_id, user_id)`.

Indexes: `contract_id`, `user_id`.

---

### `cycles`

A single time-bound period within a contract. Cycle N+1 is created and set to `active` when Cycle N transitions to `pending_resolution` ‚Äî both rows coexist during the overlap, which is how the period switcher shows current and previous cycles simultaneously.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`contract_id`** `UUID NOT NULL REFERENCES contracts(id)`
- **`cycle_number`** `INTEGER NOT NULL` ‚Äî 1, 2, 3‚Ä¶; drives the period switcher UI
- **`start_date`** `DATE NOT NULL`
- **`end_date`** `DATE NOT NULL`
- **`status`** `TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'pending_resolution', 'settled'))`

Constraints: `UNIQUE (contract_id, cycle_number)`, `CHECK (end_date > start_date)`.

---

### `habit_actions`

Individual completion slots for a participant within a cycle. Created by the service when a cycle starts ‚Äî one row per participant per `frequency` unit. A participant with `frequency = 3` gets rows with `action_number` 1, 2, 3.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`cycle_id`** `UUID NOT NULL REFERENCES cycles(id)`
- **`participant_id`** `UUID NOT NULL REFERENCES participants(id)`
- **`action_number`** `INTEGER NOT NULL` ‚Äî 1-indexed, up to the participant's `frequency`
- **`created_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`

Constraints: `UNIQUE (cycle_id, participant_id, action_number)`.

A habit action is complete when its `evidence` row has `status = 'verified'` or `status = 'auto_approved'`. The service rejects evidence submission against an already-complete action.

Indexes: `(cycle_id, participant_id)`.

---

### `evidence`

Proof submitted by a participant against a specific habit action. At least one of `photo_id` or `note` must be present.

**MVP constraint:** The service enforces one evidence row per `habit_action_id` ‚Äî a second submission is rejected if any evidence row already exists for that action. The evidence upload API accepts `habit_action_id` explicitly in the request body ‚Äî the client passes the specific action ID derived from the participant's incomplete habit actions list; the backend does not auto-assign.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`habit_action_id`** `UUID NOT NULL REFERENCES habit_actions(id)`
- **`photo_id`** `UUID` ‚Äî nullable; resolves to `evidence/{photo_id}` in object storage
- **`note`** `TEXT` ‚Äî nullable
- **`status`** `TEXT NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'verified', 'rejected', 'auto_approved'))` ‚Äî written by the service on every vote cast; set to `auto_approved` by the scheduler at settlement if zero votes were cast.
- **`submitted_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`

Constraints: `CHECK (photo_id IS NOT NULL OR note IS NOT NULL)`.

Indexes: `habit_action_id`.

---

### `votes`

An approve/reject cast by a co-participant on a piece of evidence. Voters can change their decision ‚Äî `last_updated_at` reflects the most recent change.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`evidence_id`** `UUID NOT NULL REFERENCES evidence(id)`
- **`voter_participant_id`** `UUID NOT NULL REFERENCES participants(id)` ‚Äî FK enforces only contract participants can vote
- **`decision`** `TEXT NOT NULL CHECK (decision IN ('approve', 'reject'))`
- **`created_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`
- **`last_updated_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`

Constraints: `UNIQUE (evidence_id, voter_participant_id)` ‚Äî one vote per voter per evidence item.

**Evidence status computation** (applied after every vote; result written to `evidence.status`):

- `VERIFIED` if `approvals > votes_cast / 2`
- `REJECTED` if `rejections > votes_cast / 2`
- `PENDING` otherwise (tie, or no votes yet)

Status is determined by simple majority of votes **actually cast** ‚Äî not of all eligible voters. A single vote is sufficient to decide the outcome.

**Auto-approval:** when a cycle settles, any evidence still `pending` (zero votes cast) is set to `auto_approved` by the scheduler. Evidence that already has votes retains the outcome computed from those votes.

Indexes: `evidence_id`.

---

### `cycle_resolutions`

Records the outcome of a settled cycle. Written once by the resolution service when a cycle settles. At most one row per cycle (`UNIQUE` on `cycle_id`).

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`cycle_id`** `UUID NOT NULL UNIQUE REFERENCES cycles(id)`
- **`winner_ids`** `UUID[] NOT NULL DEFAULT '{}'` ‚Äî **user IDs** (not participant IDs) of participants who are owed the forfeit. Empty if nobody is owed (everyone succeeded or exact tie). Using `user_id` rather than `participant_id` keeps historical resolutions stable if participant membership changes between cycles in the future.
- **`loser_ids`** `UUID[] NOT NULL DEFAULT '{}'` ‚Äî **user IDs** of participants who owe the forfeit. Empty if nobody owes (everyone succeeded or exact tie).
- **`forfeit`** `TEXT NOT NULL` ‚Äî snapshot of `contracts.forfeit` at resolution time; self-contained record
- **`created_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`

Outcome states:

- `winner_ids` non-empty, `loser_ids` non-empty ‚Äî losers owe the forfeit to winners
- Both empty ‚Äî nobody owes anything (everyone succeeded or exact tie)

Query membership with `WHERE ? = ANY(loser_ids)` / `WHERE ? = ANY(winner_ids)`.

Indexes: GIN on `winner_ids`, GIN on `loser_ids` ‚Äî required for `WHERE ? = ANY(winner_ids)` membership queries; B-tree indexes do not serve array containment checks.

Per-participant acknowledgment and settlement state lives in `resolution_acknowledgments` ‚Äî one row per winner or loser once they act on this resolution.

---

### `resolution_acknowledgments`

Per-participant acknowledgment and settlement tracking for a cycle resolution. One row per participant (winner or loser) per resolution. Written lazily ‚Äî a row is created when the participant first acts on the resolution.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`resolution_id`** `UUID NOT NULL REFERENCES cycle_resolutions(id)`
- **`user_id`** `UUID NOT NULL REFERENCES users(id)`
- **`acknowledged_at`** `TIMESTAMPTZ` ‚Äî nullable; set when this loser taps "I Know". Irrelevant for winners.
- **`settled_at`** `TIMESTAMPTZ` ‚Äî nullable; set when this winner marks their debt as settled. Irrelevant for losers.

Constraints: `UNIQUE (resolution_id, user_id)`.

Indexes: `resolution_id`.

A loser has fully acknowledged when their row has `acknowledged_at IS NOT NULL`. A winner has settled when their row has `settled_at IS NOT NULL`. Both are independent ‚Äî one loser acknowledging does not affect another loser's row.

The service must validate before writing that `user_id` is present in `cycle_resolutions.winner_ids` or `cycle_resolutions.loser_ids` for this resolution ‚Äî there is no FK constraint enforcing this, as membership is stored as a UUID array rather than individual rows.

---

### `pesters`

Records each instance of a winner pestering a loser to pay up. Used for rate-limiting pester sends and driving `pester` notifications.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`resolution_id`** `UUID NOT NULL REFERENCES cycle_resolutions(id)`
- **`from_user_id`** `UUID NOT NULL REFERENCES users(id)` ‚Äî the winner pestering
- **`to_user_id`** `UUID NOT NULL REFERENCES users(id)` ‚Äî the loser being pestered
- **`sent_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`

Rate limit check: `WHERE resolution_id = ? AND from_user_id = ? AND to_user_id = ? AND sent_at > now() - interval '24 hours'` ‚Äî rate limit is **per winner‚Äìloser pair**, not per resolution. A winner can pester each loser independently.

Indexes: `(resolution_id, from_user_id, to_user_id)`.

---

### `notifications`

Persistent per-user alert records. Drives the Dashboard Alert Stack and push notification delivery. Written by the service at each trigger point; never derived at query time.

- **`id`** `UUID PK DEFAULT gen_random_uuid()`
- **`recipient_user_id`** `UUID NOT NULL REFERENCES users(id)`
- **`type`** `TEXT NOT NULL CHECK (type IN ('evidence_uploaded', 'contract_invited', 'cycle_pending_resolution', 'resolution_loser', 'resolution_winner', 'pester'))`
- **`entity_id`** `UUID` ‚Äî polymorphic reference to the relevant row
- **`entity_type`** `TEXT CHECK (entity_type IN ('evidence', 'contract', 'cycle', 'cycle_resolution', 'pester'))` ‚Äî identifies which table `entity_id` points to
- **`read_at`** `TIMESTAMPTZ` ‚Äî nullable; null = unread
- **`created_at`** `TIMESTAMPTZ NOT NULL DEFAULT now()`

Type ‚Üí entity mapping:

- `evidence_uploaded` / `evidence` ‚Üí the uploaded evidence row (triggers a "Verify" alert for co-participants)
- `contract_invited` / `contract` ‚Üí the contract the user was invited to
- `cycle_pending_resolution` / `cycle` ‚Üí the cycle entering resolution (triggers a "Settle" alert)
- `resolution_loser` / `cycle_resolution` ‚Üí the resolution where the user is a loser (triggers a "Pay Up" alert)
- `resolution_winner` / `cycle_resolution` ‚Üí the resolution where the user is a winner (triggers a "You're Owed" alert on the Dashboard)
- `pester` / `pester` ‚Üí the pester row that triggered it (triggers a "Pay Up" reminder)

Indexes: `(recipient_user_id, read_at)` ‚Äî for loading the unread alert stack.
