# API

Javalin backend. OpenAPI spec generated at compile time from `@OpenApi` annotations; served at `GET /openapi.json`. Frontend types and hooks are generated by Orval ‚Äî never define API types by hand.

## Conventions

### Authentication

Two before-handlers:

- **`requireAuth`** ‚Äî verifies the SuperTokens JWT and loads the `users` row by `supertokens_user_id`. Attaches `userId` (our UUID) to the context. Used on all protected routes except registration.
- **`requireJwt`** ‚Äî verifies the SuperTokens JWT only. Does **not** load the `users` row. Used only on `POST /users` (the user row does not exist yet at that point).

The `userId` on the context is always our `users.id` UUID ‚Äî never the SuperTokens user ID, and never a caller-supplied value.

### Error responses

All errors return:

```json
{ "error": "Human-readable message" }
```

Mapped globally via `app.exception()` ‚Äî no per-handler error handling.

### File URLs

`avatarUrl` and `photoUrl` fields are resolved URLs, not raw IDs. The backend resolves them via `FileStorageService` before returning. Callers never construct storage keys.

### IDs

All IDs are UUIDs. `cycleNumber` is the integer sequence number (`cycles.cycle_number`), not a UUID.

## Users

### `POST /users` ‚Äî Register

Creates the `users` row after SuperTokens has created the auth account. Called once per signup.

**Auth:** `requireJwt` (JWT verified; no `users` row lookup)

**Request body:**

```json
{ "displayName": "Edward Lucas" }
```

**Response `201`:**

```json
{
    "id": "3fa85f64-...",
    "displayName": "Edward Lucas",
    "tag": "edward4f2a",
    "email": "edward@example.com",
    "avatarUrl": null
}
```

Backend derives `supertokens_user_id` and `email` from the JWT. Tag is auto-generated by the service (first word of display name, lowercased, stripped to alphanumeric, truncated to 8 chars, plus 4 random alphanumeric chars; retries on collision).

**Errors:**

- `409` ‚Äî a `users` row already exists for this SuperTokens account

---

### `GET /users/me` ‚Äî Get current user

**Auth:** `requireAuth`

**Response `200`:**

```json
{
    "id": "3fa85f64-...",
    "displayName": "Edward Lucas",
    "tag": "edward4f2a",
    "email": "edward@example.com",
    "avatarUrl": null
}
```

---

### `PATCH /users/me` ‚Äî Update profile

**Auth:** `requireAuth`

**Request body** (all fields optional; at least one required):

```json
{ "displayName": "Ed Lucas" }
```

MVP: `displayName` only. Avatar upload is out of scope.

**Response `200`:** updated user (same shape as `GET /users/me`).

**Errors:**

- `400` ‚Äî body is empty or `displayName` is blank

---

### `GET /users/search?tag={prefix}` ‚Äî Search users by tag

Used in the contract builder invite flow and Friends screen. Prefix-matches against `users.tag`.

**Auth:** `requireAuth`

**Query params:**

| Param | Required | Description                                 |
| ----- | -------- | ------------------------------------------- |
| `tag` | Yes      | Tag prefix to search. Minimum 2 characters. |

**Response `200`:**

```json
[
    {
        "id": "3fa85f64-...",
        "displayName": "Alex",
        "tag": "alex3b9c",
        "avatarUrl": null,
        "isContact": true
    }
]
```

- Excludes the calling user from results.
- `isContact` ‚Äî `true` if this user is already in the caller's contacts.
- Returns up to 20 results, ordered by tag ascending.

**Errors:**

- `400` ‚Äî `tag` param missing or fewer than 2 characters

## Contacts

### `GET /contacts` ‚Äî List contacts

**Auth:** `requireAuth`

**Response `200`:**

```json
[
    {
        "id": "3fa85f64-...",
        "displayName": "Alex",
        "tag": "alex3b9c",
        "avatarUrl": null
    }
]
```

Ordered alphabetically by `displayName`. Returns the full list (no pagination).

---

### `POST /contacts` ‚Äî Add a contact

**Auth:** `requireAuth`

**Request body:**

```json
{ "userId": "3fa85f64-..." }
```

The `userId` comes from a prior tag search result.

**Response `201`:**

```json
{
    "id": "3fa85f64-...",
    "displayName": "Alex",
    "tag": "alex3b9c",
    "avatarUrl": null
}
```

**Errors:**

- `400` ‚Äî `userId` is the caller's own ID
- `404` ‚Äî user not found
- `409` ‚Äî already a contact

---

### `DELETE /contacts/{contactUserId}` ‚Äî Remove a contact

Path param is the target user's ID (not the contacts row ID).

**Auth:** `requireAuth`

**Response `204`** ‚Äî no body.

**Errors:**

- `404` ‚Äî not in contacts

## Contracts

### `POST /contracts` ‚Äî Create contract

Called on FAB tap. Creates a contract with default values and the creator's participant row (`sign_status = 'drafting'`). Everything stays mutable until `POST /contracts/{contractId}/start`.

**Auth:** `requireAuth`

**Request body:** none.

**Defaults applied by the server:**

| Field       | Default        |
| ----------- | -------------- |
| `name`      | `""`           |
| `forfeit`   | `""`           |
| `period`    | `"weekly"`     |
| `startDate` | tomorrow (UTC) |
| `status`    | `"draft"`      |

**Response `201`:** full contract (see `GET /contracts/{contractId}` shape below).

---

### `GET /contracts/{contractId}` ‚Äî Get contract

Polled by both lobby screens to reflect real-time changes.

**Auth:** `requireAuth`

**Response `200`:**

```json
{
    "id": "uuid",
    "name": "Gym Bros",
    "forfeit": "A pint",
    "period": "weekly",
    "startDate": "2026-05-01",
    "status": "draft",
    "creatorId": "uuid",
    "createdAt": "2026-04-15T10:00:00Z",
    "participants": [
        {
            "id": "uuid",
            "userId": "uuid",
            "displayName": "Edward",
            "tag": "edward4f2a",
            "avatarUrl": null,
            "habit": "Go to the gym",
            "frequency": 3,
            "signStatus": "signed",
            "optedOutOfNextCycle": false
        }
    ]
}
```

`participants` excludes `removed` rows. Includes `waiting`, `drafting`, `signed`, and `declined` (so the creator can see who declined in the lobby).

Caller must be the creator or a non-removed participant.

**Errors:**

- `403` ‚Äî caller is not the creator or a participant
- `404` ‚Äî contract not found

---

### `PATCH /contracts/{contractId}` ‚Äî Update ground rules

Creator only. Any status except `cancelled` or `ended`. All fields optional; at least one required.

**Auth:** `requireAuth`

**Request body:**

```json
{
    "name": "Gym Bros 2026",
    "forfeit": "Two pints",
    "period": "biweekly",
    "startDate": "2026-05-08"
}
```

**Response `200`:** updated contract (same shape as GET).

**Errors:**

- `403` ‚Äî not the creator
- `409` ‚Äî contract is `cancelled` or `ended`

---

### `POST /contracts/{contractId}/start` ‚Äî Start contract

Creator only. Transitions `draft ‚Üí active` and creates the first cycle.

**Auth:** `requireAuth`

Validates:

- Status is `draft`
- `name` and `forfeit` are non-empty
- At least 2 `signed` participants (creator + ‚â•1 invitee)
- `startDate` ‚â§ today

**Response `200`:**

```json
{ "contractId": "uuid", "cycleNumber": 1 }
```

Client navigates to `/contract/{contractId}/1/active`.

**Errors:**

- `403` ‚Äî not the creator
- `409` ‚Äî validation failed (status not draft, fewer than 2 signed participants, name/forfeit empty, or start date not yet reached)

---

### `DELETE /contracts/{contractId}` ‚Äî Cancel contract

Creator only. Draft status only.

**Auth:** `requireAuth`

**Response `204`** ‚Äî no body.

**Errors:**

- `403` ‚Äî not the creator
- `409` ‚Äî contract not in `draft` status

## Participants

### `POST /contracts/{contractId}/participants` ‚Äî Invite participant

Creator only. Draft status only. Creates a participant row (`sign_status = 'waiting'`) and sends a `contract_invited` notification.

**Auth:** `requireAuth`

**Request body:**

```json
{ "userId": "uuid" }
```

**Response `201`:**

```json
{
    "id": "uuid",
    "userId": "uuid",
    "displayName": "Alex",
    "tag": "alex3b9c",
    "avatarUrl": null,
    "habit": null,
    "frequency": null,
    "signStatus": "waiting",
    "optedOutOfNextCycle": false
}
```

**Errors:**

- `400` ‚Äî user already a participant (any status, including `declined`)
- `403` ‚Äî not the creator
- `409` ‚Äî contract not in `draft` status, or already at 10 participants

---

### `DELETE /contracts/{contractId}/participants/{participantId}` ‚Äî Remove participant

Creator only. Draft status only. Cannot remove self.

Sets `sign_status = 'removed'`. Row is never deleted.

**Auth:** `requireAuth`

**Response `204`** ‚Äî no body.

**Errors:**

- `400` ‚Äî attempt to remove self
- `403` ‚Äî not the creator
- `404` ‚Äî participant not found
- `409` ‚Äî contract not in `draft` status

---

### `PATCH /contracts/{contractId}/participants/me` ‚Äî Update commitment

Updates `habit` and/or `frequency`. Used by both creator and invitees. Draft status only. Transitions `sign_status` from `waiting` ‚Üí `drafting` implicitly if not already there.

**Auth:** `requireAuth`

**Request body** (all fields optional; at least one required):

```json
{
    "habit": "Go to the gym",
    "frequency": 3
}
```

**Response `200`:** updated participant (same shape as in GET contract response).

**Errors:**

- `400` ‚Äî body is empty
- `409` ‚Äî contract not in `draft` status

---

### `POST /contracts/{contractId}/participants/me/sign` ‚Äî Sign contract

Locks the caller's commitment. Requires `habit` and `frequency` to already be set.

**Auth:** `requireAuth`

**Request body:** none.

**Response `200`:** updated participant.

**Errors:**

- `400` ‚Äî `habit` or `frequency` not set
- `409` ‚Äî contract not in `draft` status, or already `signed`

---

### `POST /contracts/{contractId}/participants/me/unsign` ‚Äî Unsign contract

Reverts `signed` ‚Üí `drafting` to allow editing commitment.

**Auth:** `requireAuth`

**Request body:** none.

**Response `200`:** updated participant.

**Errors:**

- `409` ‚Äî not currently `signed`, or contract not in `draft` status

---

### `POST /contracts/{contractId}/participants/me/decline` ‚Äî Decline invitation

Sets `sign_status = 'declined'`. Valid from any non-`declined` status.

**Auth:** `requireAuth`

**Request body:** none.

**Response `204`** ‚Äî no body.

**Errors:**

- `409` ‚Äî already `declined`, or contract not in `draft` status

---

### `POST /contracts/{contractId}/participants/me/opt-out` ‚Äî Opt out of next cycle

Permanently sets `opted_out_of_next_cycle = true`. Only valid on the last day of the current active cycle.

**Auth:** `requireAuth`

**Response `204`** ‚Äî no body.

**Errors:**

- `400` ‚Äî already opted out
- `409` ‚Äî not the last day of the active cycle, or contract not `active`

---

### `GET /contracts/active`

Contracts where `status = 'active'` and the caller is a `signed` participant. Carries all data needed for the dashboard contract card.

**Auth:** `requireAuth`

**Response `200`:**

```json
[
    {
        "contractId": "uuid",
        "name": "Gym Bros",
        "cycleNumber": 3,
        "startDate": "2026-04-14",
        "endDate": "2026-04-20",
        "myProgress": {
            "completed": 2,
            "pending": 0,
            "total": 3
        },
        "hasUnreviewedEvidence": true,
        "participants": [
            { "displayName": "Edward", "avatarUrl": null },
            { "displayName": "Alex", "avatarUrl": null }
        ]
    }
]
```

- `completed` ‚Äî habit actions with VERIFIED evidence (majority approved)
- `pending` ‚Äî habit actions with PENDING evidence (submitted, not yet resolved)
- `total` ‚Äî caller's `frequency` for this cycle
- `hasUnreviewedEvidence` ‚Äî `true` if any co-participant has evidence the caller hasn't voted on; drives the "REVIEW [NAME]'S PROOF" CTA
- `participants` ‚Äî all `signed` participants, for the avatar cluster on the card

---

### `GET /contracts/pending-resolution`

Contracts where the caller is a `signed` participant and a cycle has `status = 'pending_resolution'`. A contract can appear here and in `GET /contracts/active` simultaneously during the overlap period ‚Äî they represent different cycles.

**Auth:** `requireAuth`

**Response `200`:**

```json
[
    {
        "contractId": "uuid",
        "contractName": "Gym Bros",
        "cycleNumber": 2,
        "unreviewedEvidenceCount": 3,
        "participants": [
            { "displayName": "Edward", "completed": 3, "total": 3 },
            { "displayName": "Alex", "completed": 1, "total": 3 }
        ]
    }
]
```

- `unreviewedEvidenceCount` ‚Äî evidence items in this cycle the caller hasn't voted on; drives the "X reviews needed" label
- `participants` ‚Äî final verified counts for the progress summary; no `pending` since evidence submission is locked in `pending_resolution`

## Cycles

### `GET /contracts/{contractId}/cycles/{cycleNumber}` ‚Äî Get cycle

Used to establish dates, status, and (when settled) the resolution to fetch next.

**Auth:** `requireAuth`

**Response `200`:**

```json
{
    "cycleNumber": 1,
    "startDate": "2026-05-01",
    "endDate": "2026-05-07",
    "status": "active",
    "resolutionId": null
}
```

`resolutionId` is non-null only when `status = "settled"`.

**Errors:**

- `403` ‚Äî caller is not the creator or a non-removed participant
- `404` ‚Äî contract or cycle not found

## Evidence

### `GET /contracts/{contractId}/cycles/{cycleNumber}/participants/{participantId}/habit-actions` ‚Äî Get participant habit actions

Returns a participant's habit action slots for the cycle, each with their evidence nested. Used by the upload screen (to select which slot to fill) and for per-participant progress and timeline views.

**Auth:** `requireAuth`

**Response `200`:**

```json
[
    {
        "id": "uuid",
        "actionNumber": 1,
        "evidence": {
            "id": "uuid",
            "photoUrl": null,
            "note": "Went to the gym!",
            "submittedAt": "2026-05-03T08:00:00Z",
            "status": "VERIFIED"
        }
    },
    {
        "id": "uuid",
        "actionNumber": 2,
        "evidence": null
    },
    {
        "id": "uuid",
        "actionNumber": 3,
        "evidence": null
    }
]
```

**Errors:**

- `403` ‚Äî caller is not a participant
- `404` ‚Äî contract, cycle, or participant not found

---

### `GET /contracts/{contractId}/cycles/{cycleNumber}/evidence` ‚Äî List cycle evidence

All submitted evidence across all participants. Used by the active screen (per-participant progress, `hasUnreviewedEvidence`), unsettled screen, and evidence review feed.

- `hasMyVote: null` ‚Äî caller's own evidence (cannot vote)
- `hasMyVote: false` ‚Äî caller has not yet voted
- `hasMyVote: true` ‚Äî caller has voted

**Auth:** `requireAuth`

**Response `200`:**

```json
[
    {
        "id": "uuid",
        "participantId": "uuid",
        "displayName": "Alex",
        "avatarUrl": null,
        "actionNumber": 2,
        "photoUrl": null,
        "note": "Went to the gym!",
        "submittedAt": "2026-05-03T08:00:00Z",
        "status": "PENDING",
        "hasMyVote": false
    }
]
```

**Errors:**

- `403` ‚Äî caller is not a participant
- `404` ‚Äî contract or cycle not found

---

### `POST /contracts/{contractId}/cycles/{cycleNumber}/evidence` ‚Äî Submit evidence

**Auth:** `requireAuth`

**Request body:**

```json
{
    "habitActionId": "uuid",
    "note": "Went to the gym!",
    "photoId": "uuid"
}
```

At least one of `note` or `photoId` required. `photoId` is a UUID returned from a prior file upload (presigned URL flow ‚Äî TBD).

**Response `201`:** the created evidence item (same shape as list item above, with `hasMyVote: null`).

**Errors:**

- `400` ‚Äî neither `note` nor `photoId` provided, or `habitActionId` does not belong to the caller
- `409` ‚Äî evidence already submitted for this action, or cycle is not `active`

---

Used by the evidence detail screen. Includes full vote breakdown.


**Errors:**

- `403` ‚Äî caller is not a participant

## Votes

### `POST /contracts/{contractId}/cycles/{cycleNumber}/evidence/{evidenceId}/votes` ‚Äî Cast or update vote

Upsert semantics ‚Äî voter can change their decision. Voting is allowed when cycle status is `active` or `pending_resolution`. Closed once `settled`.

**Auth:** `requireAuth`

**Request body:**

```json
{ "decision": "approve" }
```

**Response `200`:**

```json
{
    "voterParticipantId": "uuid",
    "decision": "approve",
    "updatedAt": "2026-05-03T09:30:00Z"
}
```

**Errors:**

- `400` ‚Äî caller is the evidence submitter (cannot vote on own evidence)
- `403` ‚Äî caller is not a signed participant
- `404` ‚Äî evidence not found
- `409` ‚Äî cycle is `settled`
