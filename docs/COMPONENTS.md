# Shared Components

Shared UI components live in `app/components/`. This file tracks what exists and what each component is for.

All components must follow the design system defined in [product/DESIGN.md](../product/DESIGN.md): neo-brutalist aesthetic, hard drop shadows, heavy borders, zero border radius, Plus Jakarta Sans typography, all-caps labels.

## Component Entry Format

Each entry should cover:

- **Purpose** — what problem it solves, when to use it
- **Variants** — distinct visual/behavioural states
- **Props** — key inputs (conceptual, not TypeScript — the code is the source of truth once implemented)
- **Design notes** — any specific design system tokens or patterns from DESIGN.md that apply
- **File** — path once implemented (e.g. `app/components/ContractCard.tsx`)
